# Bug Fixes Summary (2026-03-30)

## Issues Fixed

### 1. RateLimiter Missing Methods
**File**: `orchestrator/rate_limiter.py`

**Problem**: `RateLimiter` class was missing `check()`, `release()`, and `record()` methods that were being called in `engine.py`.

**Fix**: Added backward compatibility methods as no-ops:
```python
def check(self, tenant: str, model: str, tokens: int) -> None:
    pass  # No-op for now

def release(self, tenant: str, model: str, tokens: int) -> None:
    pass  # No-op for now

def record(self, tenant: str, model: str, tokens: int) -> None:
    pass  # No-op for now
```

### 2. CLI `orch` Not Defined
**File**: `orchestrator/cli.py`

**Problem**: `_print_results()` function referenced `orch` variable that wasn't in scope.

**Fix**: 
- Changed function signature to accept `orch` as optional parameter
- Updated all call sites to pass `orch` parameter

### 3. TaskResult Missing `success` Property
**File**: `orchestrator/models.py`

**Problem**: `TaskResult` class was accessed via `.success` but didn't have this attribute.

**Fix**: Added property for backward compatibility:
```python
@property
def success(self) -> bool:
    """Backward compatibility: success = completed status and score > 0."""
    return self.status == TaskStatus.COMPLETED and self.score > 0.5
```

### 4. Invalid Model: `microsoft/phi-4-reasoning-plus`
**File**: `orchestrator/models.py`

**Problem**: Model `microsoft/phi-4-reasoning-plus` doesn't exist on OpenRouter (404 error).

**Fix**: Changed to use `openai/o3-mini` for reasoning tasks:
```python
PHI_4_REASONING = "openai/o3-mini"  # Use o3-mini for reasoning
```

### 5. Removed Model References
**Files**: Multiple files

**Problem**: Code referenced models that were removed from the enum.

**Fix**: Updated all references:
- `GEMINI_2_5_FLASH_LITE` → `GEMINI_FLASH_LITE`
- `MISTRAL_NEMO`, `MISTRAL_SMALL_3_1` → `PHI_4`, `GEMMA_3_27B`
- `MINIMAX_TEXT_01` → `LLAMA_3_3_70B`
- `KIMI_K2_5` → `PHI_4`

### 6. MODEL_MAX_TOKENS Cleanup
**File**: `orchestrator/models.py`

**Problem**: `MODEL_MAX_TOKENS` dictionary had entries for models that no longer exist.

**Fix**: Updated to only include existing models (28 total).
